Table Content Context Menu (no Table Style applied)

 

 Although most of the option in this section and the next are the same, all menu options must be tested in each context. In this section, test with a table that DOES NOT USE a Table Style.

 

Invoking this Context Menu:

 

 With no selection, simply right-click anywhere inside a table that does not use a Table Style in the XML Editor. This menu is an expanded version of the XML Editor's standard right click menu when there no selection.

 

Menu Test Cases

 

 Menu items are correct

 

 Tooltips are correct

 

 Spelling and grammar for items and tooltips are correct

 

 No duplicate accelerators

 

 All accelerators function

 

 Keyboard can navigate and select options

 

 Menu is easy to use and understand
   
 

Context Menu Controls (Tooltips in parentheses):Table specific items only

 

 Insert: Invokes a sub-menu of Insert options

 

 Table (Insert Table): (Inactive if multiple cells are selected) Invokes the Insert Table dialog and inserts a table in the current cell.

 

 Columns to the Left (Insert new column to the right of the selected column): Inserts as many columns as are included in the current selection to the left of the leftmost selected column.

 

 Columns to the Right (Insert new column to the right of the selected column): Inserts as many columns as are included in the current selection to the right of the rightmost selected column.

 

 Rows Above (Insert new row above the selected row): Inserts as many rows as are included in the current selection above the topmost selected row.

 

 Rows Below (Insert new row below the selected row): Inserts as many rows as are included in the current selection below the bottommost selected row.

 

 Delete: Invokes a sub-menu of Delete options

 

 Table (Delete table): Deletes the entire table element.

 

 Column (Delete selected column(s)): Deletes the col elements based on the current selection or insertion point.

 

 Row (Delete selected row(s)): Deletes the tr elements based on the current selection or insertion point.

 

 Select: Invokes a sub-menu of Select options

 

 All Cells (Select all cells): Selects all of the th and td elements in the entire table.

 

 Table (Select table): Selects the table element.

 

 Column (Select column): Although this appears to select the col elements, it currently (through V9) selects all of the th and td elements based on the current selection or insertion point.

 

 Rows (Select row): Although this appears to select the tr elements, it currently (through V9) selects all of the th and td elements based on the current selection or insertion point.

 

 Cell (Select cell): Selects the th or td element based on the current insertion point.

 

 Sort Rows: Invokes a sub-menu which lets you sort the selected table rows. Rows will sort based on the content of the leftmost selected cells for LTR languages and the rightmost selected cells for RTL languages.

 

 Ascending (Sort the selected rows in ascending order): Sorts the selected rows in ascending order (symbols, 0-9, A-Z).

 

 Descending (Sort the selected rows in descending order): Sorts the selected rows in descending order (Z-A, 9-0, symbols)

 

 Clear Cells (Clear selected cells): Clears the content from the selected cell

 

 Merge Cells (Merge selected cells): Active only if you have two cells selected. Launches the Merge Cells dialog

 

 Split Cells... (Split selected cells): Invokes the Split Cells dialog

 

 Cell Content Style... (Select style for cell content): Invokes the Table Cell Content Style dialog which lets you select the style of the cell contents for the selected cells.

 

 Cell Class: Invokes the Cell Class sub-menu

 

 Create Style Class (Create a new CSS style class): Launches the Create Style dialog

 

 Edit Style Class (Edit the CSS style): Opens the associated CSS and selects the element or element.Class

 

 Reset Style Class (Reset style class): Active only if a class is selected. Removes the class from the tag.

 

 Reset Local Cell Formatting (Remove potential local formatting set on the cells): Resets the local cell formatting of the selected table cells

 

 Table Style: Invokes the Table Style sub-menu

 

 Edit Style (Edit the Style document): Active only if a Table Style is selected. Opens the associated Table Style in the Table Style Editor.

 

 Reset Table Style (Reset Table Style): Active only if a Table Style is selected. Removes the Table Style association from the Table.

 

 (Table Styles List): All Table Styles in the project should appear listed here. The Table Style will have a check next to it if it is applied to the current table.

 

 Table Class: Invokes the Table Class sub-menu

 

 Create Style Class (Create a new CSS style class): Launches the Create Style dialog

 

 Edit Style Class (Edit the CSS style): Opens the associated CSS and selects the element or element.Class

 

 Reset Style Class (Reset style class): Active only if a class is selected. Removes the class from the table tag.

 

 (Class List): Lists all of the table classes in the linked stylesheet (or Master Stylesheet). The class will have a check next to it if it is applied to the current table.

 

 Table Properties (Run the Table Properties Dialog): Invokes the Table Properties dialog for the current table

Block Bars, Span Bars, Tooltips and Caret Tag
 

Block/Span Bar and Tooltip:
  Make sure block bars and span bars are enabled using the and buttons in the XML Editor's lower toolbar. Block and span bars are used to see and select a tag and to get easy access to the element's context menu.
  Hovering over the block/span bar should reveal a tooltip displaying the tag and any attributes set on it
  Clicking the block/span bar will select the element
  Right-clicking the block/span bar will invoke the context menu for the element
 

Caret Tag Neighborhood (inline elements only):
  Make sure "Show Caret Tag Neighborhood" is enabled (Toolstrip interface: !View | Show; or click the "Show" drop-down menu in the XML Editor's local toolbar and enable "Show Caret Tag Neighborhood"). Place the insertion point within the same paragraph level tag as the element to reveal the caret tag neighborhood.
  Caret tag neighborhood displays the correct tag
  Clicking the caret tag will select the element
  Right-clicking the caret tag will invoke the context menu for the element
   
 

Conditional Tags
 

Applying a condition tag to the element:
  Select the element and invoke the Condition Tags dialog
  Right-click the element's block/span bar and select Conditions… from the context menu
  Right-click the element's caret tag neighborhood bar and select Conditions… from the context menu
  Right-click the selected element in the XML Editor and select Conditions… from the context menu
  Select the element and press CTRL+Shift+C
  Toolstrip interface: Select the element and then !Format | Conditions…
  Ribbon interface: Select the element and then !Home | Conditions
  In the Condition Tags dialog, select condition tags
  Select one or more condition tags to apply to the element. Upon clicking OK, the element's block/span bar will be colored to display the applied condition.
  If available, also test the Exclude Action option. Upon clicking OK, the element's block/span bar will be colored to display the applied condition.
 

Removing a condition tag from the element:
  Once conditions are applied, select the element and invoke the Condition Tags dialog again and deselect all applied conditions.
  Verify all conditions are removed in the markup. There should not be an empty attribute (You should not see "MadCap:conditions=" "" in the markup).
  Verify the Exclude Action markup is also removed if it was set (You should not see "MadCap:excludeAction="unbind"" in the markup).

Structure Bars
  The structure bars for tables have been improved for V7 in the XML Editor.
   
 

Table Rows/Tag Bars:
  The table row bars have been merged with the regular tag block bars on the left of the editor. To turn these on, you can click the button for showing tag block bars .
  Starting with this version, you can see all of the main table row tags, whether you click inside a table or not. You can see additional tags when you click in certain cells.
  The main <table> tag can be seen. This tag represents the entire table.
  Within that tag you can see the <caption>, <thead>, <tbody>, and <tfoot> tags. These tags represent any table captions, header rows, regular table body rows, and footer rows, respectively.
  Within those tags you can see the <tr> tags, which represent the individual rows in the table.
  Within those tags you can see <th> or <td> tags when you click in different cells. These tags represent content for table headers or regular cell text, respectively.
  And finally, within those tags you may see other types of tags such as <p>, <ul>, or <li> (paragraphs, lists), depending on the types of content you add to those rows.
   
 

Table Columns/Span Bars:
  The table column bars are now grouped with the span bars, being displayed above them in the XML Editor. To turn these on, you can click the button for showing span bars . The old button for showing table column bars has been removed.
